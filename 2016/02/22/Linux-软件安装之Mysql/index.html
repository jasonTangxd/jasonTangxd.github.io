Linux--软件安装之MySQL

2016-02-22
Mysql Linux
  1. 准备
  2. 卸载Mysql
  3. 安装Mysql
  4. 初始化mysql

　　Mysql的安装，第一次自己去折腾的时候倒挺麻烦的，下载哪一个版本的mysql都不确定。之后在DBA的帮助下，很顺畅的就完成了，在这里呢总结一下安装流程：

准备

可以去官网下载mysql数据库，我这里下载的是：mysql-5.7.9-1.el6.x86_64.rpm-bundle.tar，点击这里可下载。

  • 检测系统中是否存在mysql，存在就卸载

卸载Mysql

  • 卸载Mysql

  • 如下信息，就需要–nodeps忽略依赖

安装Mysql

  1. 准备阶段
    1.1. 上传我们下载好的mysql tar文件。
    2.2. 然后使用命令：tar -xvf ；发现mysql包含了很多的rpm文件。
    3.3. 我们需要使用rpm命令从上面的rpm文件一个一个的安装。

  2. 安装第一个mysql-community-common-5.7.9-1.el6.x86_64.rpm，第一个顺利完成。

  3. 安装第二个mysql-community-server，发现需要依赖包mysql-community-client

  4. 于是我们安装mysql-community-client，发现需要依赖包lib，

  5. 然后安装lib成功，安装client也成功了,目前到这里还是比较顺利。

  6. 然后再次尝试安装server,发现还需要很多的依赖包。晕菜了，这么多，这个可以按照上面的方法逐步安装,是可以得，博客中就不讲解了，换成使用yum命令。

  7. 使用yum命令安装一下server，最下图表示安装完成！

初始化mysql

  1. mysql_install_db

    mysql_install_db is deprecated. Please consider switching to mysqld –initialize，mysql_install_db已弃用，使用新命令。

  2. mysqld --initialize命令不现实任何信息，表示是个好的结果，去var/log/mysqld.log查看一下日志，这里面记录了mysql的初始密码

  3. 尝试启动一下mysql

  4. 启动失败，这时候需要去/var/lib修改一下mysql的所属权限

  5. 重新启动mysql数据库，然后设置密码，就可以登陆了。
