Linux--用户和组管理

2016-01-18
Linux
  1. 配置文件
  2. 用户管理
  3. 用户组管理
  4. 权限管理
    1. 授权
    2. 修改所属

　　在Linux系统中，用户信息及密码，组信息及其密码，都保存在不同的文件中，

  1. 用户信息（/etc/passwd），
  2. 用户密码（/etc/shadow），
  3. 用户组帐号（/etc/group），
  4. 用户组密码（/etc/gshadow）。

用户和组是多对多的关系，即一个用户可以属于多个组，一个组也可以包含多个用户。

配置文件

  1. /etc/passwd
    /etc/passwd：保存了所有用户信息，密码使用了x表示(不是明文，加密了的)。

    root:x:0:0:root:/root:/bin/bash
    表示：用户帐号|用户密码|用户ID|用户组ID|用户名全称|用户主目录|用户所使用的shell。
    该文件默认就会初始化很多用户，当shell等于/sbin/nologin时就表示该用户不能登陆。

  2. /etc/shadow

    第二位为用户密码，使用了MD5加密算法，超级用户才拥有该文件读权限。

  3. /etc/group

    分别表示：用户组名称 x 用户组ID 用户组成员列表

  4. /etc/gshadow
